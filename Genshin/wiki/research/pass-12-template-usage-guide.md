# Pass 12 Template Usage Guide

## Purpose

Future character passes should use `wiki/entities/characters/_character-template-v2.md` as the standard structure for Character Foundation Pages. The template is designed to keep character pages canon-aware, source-status explicit, OOC-safe, relationship-safe, and usable across many fanfic genres without turning the global wiki into a project-specific outline.

## When to Use Each Variant

| Character situation | Variant | Target level | Notes |
|---|---|---|---|
| playable with voice source | Standard Level 2 | Level 2 | normal foundation batch |
| high-risk with verified source IDs | Claim-Mapped | Level 4 | after review only |
| source-light character | Source-Light | Level 1 / guarded Level 2 | no hard canon expansion |
| missing source package | Blocked | Level 1 blocked | do not upgrade |

## How Future Foundation Batches Should Work

- Read the character page if it exists.
- Read the voice-line file if it exists.
- Use character intelligence only as an indirect source.
- Fill only what source coverage supports.
- Mark uncertain sections as `source_missing`, `source_light`, or `indirect_source_only`.
- Do not add character-story claims unless provenance exists.
- Do not claim-map unless the pass explicitly does Level 4 work.
- Keep genre notes reusable across projects.
- Do not write plot, outline, divergence timelines, or premise-specific story plans.

## Required Sections for Level 2

- Frontmatter
- Audit Status
- Summary
- Source Coverage Notes
- Canon Identity
- Canon Personality
- Strongly Implied Traits
- Theory / Interpretation
- Headcanon / Fanon
- OOC Risk Register
- Voice Guide
- Relationships
- Power / Lore Constraints
- Genre Usage Notes
- Safe Uses
- Unsafe Uses
- Sources
- Maintenance Notes

## Required Sections for Blocked Characters

- Audit Status
- Summary with warning
- Source Coverage Notes
- Blocked Warnings
- Safe Uses
- Unsafe Uses
- Sources
- Maintenance Notes

Blocked pages may omit broad personality, relationship, and voice expansion when the relevant direct source is missing. If a section is retained for structure, it must state that the material is blocked or source-missing.

## Common Mistakes to Avoid

- Using a folder path as a source.
- Treating character_intelligence as primary.
- Upgrading theory because it is popular.
- Copying Fandom summaries as canon.
- Adding plot-specific divergence.
- Adding fanfic outline.
- Turning genre notes into chapter plan.
- Pasting long copyrighted text into source pages.
- Creating placeholder source IDs for missing files.
- Generalizing one verified evidence field into a broader personality or lore claim.
